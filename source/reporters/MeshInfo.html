MeshInfo

A Reporter object for tracking mesh information, such as the number of elements, nodes, or degrees-of-freedom. The "items" parameter controls the items computed.

Example Input Syntax

The following input file snippet demonstrates the use of the MeshInfo object.

[Reporters/mesh_info]
  [Reporters/mesh_info]
    type = MeshInfo
  []
[]
(moose/test/tests/reporters/mesh_info/mesh_info.i)

Input Parameters

  • execute_onTIMESTEP_ENDThe list of flag(s) indicating when this object should be executed, the available options include NONE, INITIAL, LINEAR, NONLINEAR, TIMESTEP_END, TIMESTEP_BEGIN, FINAL, CUSTOM.

    Default:TIMESTEP_END

    C++ Type:ExecFlagEnum

    Options:NONE, INITIAL, LINEAR, NONLINEAR, TIMESTEP_END, TIMESTEP_BEGIN, FINAL, CUSTOM

    Description:The list of flag(s) indicating when this object should be executed, the available options include NONE, INITIAL, LINEAR, NONLINEAR, TIMESTEP_END, TIMESTEP_BEGIN, FINAL, CUSTOM.

  • itemsThe iteration information to output, if nothing is provided everything will be output.

    C++ Type:MultiMooseEnum

    Options:num_dofs, num_dofs_nonlinear, num_dofs_auxiliary, num_elements, num_nodes, num_local_dofs, num_local_dofs_nonlinear, num_local_dofs_auxiliary, num_local_elements, num_local_nodes, local_sidesets, local_sideset_elems, sidesets, sideset_elems

    Description:The iteration information to output, if nothing is provided everything will be output.

  • prop_getter_suffixAn optional suffix parameter that can be appended to any attempt to retrieve/get material properties. The suffix will be prepended with a '_' character.

    C++ Type:MaterialPropertyName

    Options:

    Description:An optional suffix parameter that can be appended to any attempt to retrieve/get material properties. The suffix will be prepended with a '_' character.
